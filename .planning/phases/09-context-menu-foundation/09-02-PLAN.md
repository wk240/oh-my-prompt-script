Context menu click handler pattern:
```typescript
chrome.contextMenus.onClicked.addListener((info: chrome.contextMenus.OnClickData, tab?: chrome.tabs.Tab) => {
  if (info.menuItemId === 'convert-to-prompt') {
    if (!info.srcUrl) {
      console.warn('[Oh My Prompt] No srcUrl in click data')
      return
    }
    // Validate URL is http/https
    if (!info.srcUrl.startsWith('http://') && !info.srcUrl.startsWith('https://')) {
      console.warn('[Oh My Prompt] Invalid URL type:', info.srcUrl)
      return
    }
    // Store captured URL
    chrome.storage.local.set({
      _capturedImageUrl: {
        url: info.srcUrl,
        capturedAt: Date.now(),
        tabId: tab?.id
      }
    })
    console.log('[Oh My Prompt] Captured image URL:', info.srcUrl, 'from tab:', tab?.id)
  }
})
```

```typescript
// Phase 9: Handle context menu click - capture image URL (MENU-03, D-05)
chrome.contextMenus.onClicked.addListener((info: chrome.contextMenus.OnClickData, tab?: chrome.tabs.Tab) => {
  if (info.menuItemId === 'convert-to-prompt') {
    if (!info.srcUrl) {
      console.warn('[Oh My Prompt] No srcUrl in click data')
      return
    }

    // Double-check URL type (targetUrlPatterns should handle this, but validate anyway)
    if (!info.srcUrl.startsWith('http://') && !info.srcUrl.startsWith('https://')) {
      console.warn('[Oh My Prompt] Invalid URL type (not http/https):', info.srcUrl)
      return
    }

    // Store captured URL for Phase 11 Vision API processing (D-05, D-08)
    chrome.storage.local.set({
      [CAPTURED_IMAGE_STORAGE_KEY]: {
        url: info.srcUrl,
        capturedAt: Date.now(),
        tabId: tab?.id // Store tab ID for Phase 12 (insert vs clipboard decision)
      }
    })

    console.log('[Oh My Prompt] Captured image URL:', info.srcUrl, 'from tab:', tab?.id)
  }
})
```

Handler logic:
1. Check menuItemId matches 'convert-to-prompt'
2. Validate srcUrl exists (OnClickData property)
3. Double-check URL is http/https (defensive validation)
4. Store URL + metadata in chrome.storage.local under CAPTURED_IMAGE_STORAGE_KEY
5. Log capture for debugging

Note: Use CAPTURED_IMAGE_STORAGE_KEY constant (import added in Task 1) for storage key.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Chrome Browser → Service Worker | Context menu click data from browser |
| Service Worker → chrome.storage.local | URL storage write |
| User → External Website → Image URL | URL comes from untrusted web source |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03 | Spoofing | targetUrlPatterns | mitigate | Filter to http/https only (prevents data:, blob: URLs with embedded content) |
| T-09-04 | Tampering | info.srcUrl | mitigate | URL stored as string only, no execution or rendering - safe for Vision API pass-through |
| T-09-05 | Tampering | CAPTURED_IMAGE_STORAGE_KEY | mitigate | Underscore prefix prevents collision with StorageSchema keys per D-08 |
| T-09-06 | Information Disclosure | stored URL | accept | URL stored locally only, no external transmission in Phase 9 (Phase 11 will send to Vision API) |
</threat_model>
