# Phase 02: Lovart Integration - Plan 01

## Objective

Establish input detection infrastructure and Lovart style extraction. This plan creates the foundation for detecting Lovart's dynamic input element and capturing Lovart's visual styles for UI coordination.

## Context

- @`.planning/phases/02-lovart-integration-content-script/02-RESEARCH.md` - MutationObserver pattern, debounce config
- @`.planning/phases/02-lovart-integration-content-script/02-UI-SPEC.md` - Lovart CSS extraction strategy
- @`src/shared/constants.ts` - LOVART_DOMAIN, STORAGE_KEY
- @`src/shared/types.ts` - Prompt, Category interfaces

## Tasks

### Task 1: Implement InputDetector Class

**Action:**
1. Create `src/content/input-detector.ts`
2. Implement MutationObserver with config per UI-SPEC:
   - `childList: true`, `subtree: true`, `attributes: false`
   - Debounce: 100ms (per RESEARCH.md)
3. Add selector discovery logic with fallback patterns:
   ```typescript
   const INPUT_SELECTORS = [
     'textarea[placeholder*="prompt"]',
     'textarea[placeholder*="提示"]',
     '.input-area textarea',
     '[data-testid="prompt-input"]',
     // Placeholder - will be refined after Lovart analysis
   ]
   ```
4. Implement SPA navigation detection (url change observer)
5. Export `InputDetector` class with `start()`, `stop()`, `onInputDetected` callback

**Implements:** CORE-01 prep, D-01 positioning prep
**Verification:**
- Unit test: InputDetector finds mock textarea element
- Console log shows `[Lovart Injector] Input detected:` when element found
- Navigation detection triggers on simulated URL change

### Task 2: Implement LovartStyleExtractor Utility

**Action:**
1. Create `src/content/style-extractor.ts`
2. Implement `extractLovartButtonStyle()` function:
   - Query Lovart button element at runtime
   - Extract `backgroundColor`, `borderRadius`, `boxShadow`, `color` via `getComputedStyle()`
3. Define `DEFAULT_STYLE` fallback per UI-SPEC Section 3:
   ```typescript
   const DEFAULT_STYLE = {
     backgroundColor: '#f5f5f5',
     borderRadius: '8px',
     boxShadow: 'none',
     color: '#666',
   }
   ```
4. Export `LovartStyleConfig` interface for type safety

**Implements:** D-04 (icon color), D-08 (Lovart-native), D-09 (manual CSS replication)
**Verification:**
- Function returns LovartStyleConfig object
- Default style fallback used when Lovart button not found
- Exported type compiles without errors
