# 本地备份弹窗设计

## 概述

将现有的 inline SyncSettingsPanel 改为弹窗形式，刷新按钮触发备份+刷新操作，未设置文件夹时引导用户选择。

---

## 组件变更

### 新增

- **BackupSettingsDialog.tsx** — 备份设置弹窗组件

### 修改

- **Header.tsx** — 刷新按钮点击逻辑变更
- **App.tsx** — 移除 SyncSettingsPanel，管理 BackupSettingsDialog 状态

### 删除

- **SyncSettingsPanel.tsx** — inline 面板不再使用

---

## 刷新按钮行为流程

```
点击刷新按钮
  ↓
检查 folder handle 是否存在且有权限
  ↓
├─ 有权限 → 执行备份 → 刷新数据 → toast 成功
│
└─ 无权限/不存在 → 打开 BackupSettingsDialog
```

## BackupSettingsDialog 布局

### 状态一：未选择文件夹

| 元素 | 内容 |
|------|------|
| 标题 | 本地备份设置 |
| 描述 | 选择文件夹以启用备份，数据变更时自动同步 |
| 按钮 | [选择文件夹并启用] |

### 状态二：已选择但未启用

| 元素 | 内容 |
|------|------|
| 标题 | 本地备份设置 |
| 状态 | 同步已禁用 |
| 文件夹 | `{文件夹名}` |
| 描述 | 文件夹已保存，启用后将自动复用 |
| 按钮 | [启用备份] [更换文件夹] |

### 状态三：已启用

| 元素 | 内容 |
|------|------|
| 标题 | 本地备份设置 |
| 状态 | 已启用 |
| 文件夹 | `{文件夹名}` |
| 上次备份 | `{格式化时间}` |
| 按钮 | [立即备份] [更换文件夹] [禁用] |

## 备份文件

### 文件名

`oh-my-prompt-script-backup.json`

### 内容结构

```json
{
  "version": "1.0.0",
  "userData": {
    "prompts": [...],
    "categories": [...]
  },
  "backupTime": "2026-04-20T14:30:52.000Z"
}
```

---

## 复用模块

| 模块 | 用途 |
|------|------|
| sync-manager.ts | folder handle 管理、同步状态、启用/禁用逻辑 |
| file-sync.ts | 文件读写操作 |
| IndexedDB 存储 | folder handle 持久化 |
| syncToLocalFolder | 备份写入函数（需调整文件名） |

---

## 错误处理

| 场景 | 处理 |
|------|------|
| 文件夹权限失效 | 弹窗提示"权限已失效，请重新选择文件夹" |
| 备份写入失败 | toast 提示"备份失败，请检查文件夹权限" |
| 用户取消选择文件夹 | 无操作，弹窗保持打开 |
